# Risks And Guardrails

记录 SOP 项目已知风险、禁区和过期信息。

## 安全禁区

- 不提交 AK、token、cookie、session、浏览器 profile。
- 不提交 `.local-data/` 中的真实采集快照或导出表格。
- 不提交真实用户数据、客户数据或私密在线文档内容。
- 不绕过 1688 登录、滑块、验证码或安全风控。

## 外部服务

| Service | 发送数据 | 风险 | 当前处理 |
|---|---|---|---|
| 1688 Web 页面 | 搜索词、类目、页面访问行为 | 登录/风控/验证码/账号安全 | 使用真实浏览器人工登录；触发验证时停止，不绕过 |
| 1688 AK/API | AK 和查询参数 | 凭证泄露、权限不足 | 仅从本地环境读取，不写入仓库 |
| 在线文档同步 | 导出结果和人工复核字段 | 私密数据泄露 | 当前仅预留接口，未自动同步 |

## 已知风险

| Risk | 影响 | 当前处理 | 更新时间 |
|---|---|---|---|
| 1688 类目字典不是完整官方实时字典 | 类目筛选可能不全 | 使用本地种子和可追溯状态，后续补官方/页面采集字典 | 2026-06-16 |
| 店雷达部分字段依赖其自有数据源 | 店铺粉丝、卖家服务等不能从 1688 列表稳定获取 | 接口预留，不伪造结果；用户已确认不需要的 7日增长率/采购集中率等筛选入口已移除 | 2026-06-17 |
| 详情字段列表页不可信 | 运费、品退率、发货率等列表字段可能缺失或不准 | 高潜商品进入详情页核验并记录字段级证据 | 2026-06-16 |
| 真实采集可能触发风控 | 账号被要求验证，采集中断 | 不绕过验证；提示人工完成或停止任务 | 2026-06-16 |
| 反复刷新会加重 1688 风控 | 账号、浏览器、IP 可能继续被拒绝访问 | 命中滑块/访问拒绝后返回 `security_verification_required`，暂停任务并要求人工接管，不自动重试 | 2026-06-17 |
| RPA 节奏过机械会触发风控 | 测试机出现“访问被拒绝” | 默认启用 `TAG_COLLECT_RPA_PACING=human`，随机等待、分段滚动，并将 Web 默认每词采集/详情核验收紧到 5 条 | 2026-06-17 |
| GitHub 公开爬虫方案含反爬绕过内容 | 可能违反平台规则并增加封控风险 | 只参考人工浏览后读取已加载页面、候选池和字段导出等产品模式；不引入绕过验证码/反爬逆向代码 | 2026-06-17 |
| 详情核验失败被误读为整体成功 | 列表采集成功但运费、品退率、发货率未可信核验 | 返回并导出 `automation_state`；详情风控时标记 `paused/manual_handoff`，页面黄色提示，导出写入暂停原因 | 2026-06-17 |
| 截图新增筛选依赖页面 DOM | 所在地、商家特色、经营模式等下拉筛选可能因 1688 页面结构变化而无法点击 | 生成 `native_filters` 并记录 clicked/not_found/click_failed；店铺商品数暂按预留字段处理，不伪造过滤成功 | 2026-06-17 |
| 评价口碑字段需详情页可见 | 商品星级、评价标签、好评率和评价数可能因登录态或懒加载不可见 | 详情页核验后才重评估；提取不到则保留待核验/失败状态，导出字段级记录 | 2026-06-17 |
| 外层网关 502 与本地服务故障混淆 | 误以为 tag_collect 页面或项目 skill 损坏 | `tag_collect` 本地服务不生成 502；若 workflow/AIDock 或反代返回 502，先查网关、端口转发和模型请求链路 | 2026-06-17 |
