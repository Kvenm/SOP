# 1688 低频真实采集测试计划

> 当前计划从 2026-07-01 开始执行。真实采集只做低频验证，不做压测，不使用账号池；出现滑块、验证码、访问拒绝、登录异常时立即停止当天真实采集。

## 1. 测试目标

- 验证选择本系统类目后，1688 侧是否进入对应的一二三级类目，而不是在搜索框泛搜。
- 验证每个筛选条件是否真实生效，不能把未执行、未找到、未核验的条件当作已筛选。
- 验证导出字段是否来自真实列表页或详情页核验，不能把来源未知的数据标成真实数据。
- 逐批标注已测字段，下一轮优先测试 `未测` 和 `异常待复测` 项。

## 2. 测试原则

- 使用 1 个正常真实账号和真实 Chrome 登录态，暂不使用账号池。
- 每个时间段只测 3-5 个类目，每个类目采集 3-5 条商品。
- 类目之间间隔 1-3 分钟；商品详情页之间间隔 15-40 秒。
- 每轮最多导出 1 个 Excel，导出后检查字段真实性说明。
- 只把 `数据模式=真实数据` 且有真实来源/核验记录的行作为有效测试结果。
- 每个时间段测试完成后，必须同时更新排期表状态、字段台账状态、测试日期和结果记录。
- 出现风控后停止，不继续刷新、重试或换方式硬跑。

## 3. 状态标记

| 状态 | 含义 | 下一步 |
|---|---|---|
| 未测 | 尚未执行 | 按计划继续测试 |
| 通过 | 类目进入、字段采集、导出均符合预期 | 标注完成，不重复测试 |
| 异常待复测 | 有问题，已记录原因 | 修复后低频复测 |
| 风控暂停 | 触发登录/滑块/验证码/访问拒绝 | 停止当天真实采集 |
| 不适用 | 当前类目或页面没有该字段 | 记录原因，不算失败 |

## 7. 补测规则

- 每天测试前先查看字段测试台账，优先选择 `异常待复测`，其次选择 `未测`。
- 已标记 `通过` 的字段不重复测，除非相关代码改动影响该字段。
- 触发风控的字段当天不继续测，标记 `风控暂停`，第二天或账号状态恢复后再测。
- 修复代码后，只复测对应字段编号和关联导出字段，不扩大采集量。
